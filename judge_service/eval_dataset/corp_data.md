# Справочные формулировки для тестовых диалогов (синтетика)

Этот файл задаёт **согласованные** фразы для ручной разметки датасета под критерии судьи (`compliance`, `sales_skills`, `knowledge`). Не юридическая оферта — только для локальных eval-фикстур.

## Комплаенс / скрипт

- Корректное приветствие: представиться как сотрудник **OZON** кириллицей, без «Озон» в кривых падежах.
- Избегать слова **«банк»** в речи менеджера (говорить «сервис», «расчётное обслуживание» и т.д.).
- Не использовать шаблоны вроде «Удобно говорить?» / «Уделите пару минут?» после того как клиент уже ответил.
- Ранние квалификационные вопросы («Вы ИП или ООО?», система налогообложения) — только если это следует из **scenario_description**.

## Продажные навыки

- Перед серией вопросов — **фраза присоединения** (эмпатия).
- Вопросы — преимущественно **открытые или альтернативные**, не цепочка закрытых «да/нет».
- Перед переходом к следующему блоку — **резюме** сказанного клиентом.

## Знания (РКО)

- Если тариф/лимиты **не назывались** — в диалоге допустимо говорить «разберём тариф на следующем шаге» без цифр.
- Для негативного кейса «ошибка в цифрах» — заведомо неверная сумма (например «990 рублей в месяц за всё»), не совпадающая с тем, что обычно в материалах.

## Персоны сценариев (из БД)

Имена сценариев вида `RKO / <persona> / level <1–4>` соответствуют строкам в `training_scenarios` после `init.sql`.
